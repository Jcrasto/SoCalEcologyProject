# SoCal Oak Explorer

An interactive field guide and distribution explorer for the 8 native oak species
of Southern California, built for the [ReOak California](https://reoakcalifornia.com)
reforestation initiative.

---

## Overview

SoCal Oak Explorer combines:

- A **React + Leaflet** frontend with species cards, tabbed detail pages (overview,
  ID guide, ecology, map, photos), GBIF density tile overlays, and a county
  distribution bar chart.
- A **FastAPI** backend backed by SQLite (via SQLAlchemy) that serves occurrence
  records, species metadata, and photos.
- **Python ETL scripts** to pull research-grade observations from GBIF and
  iNaturalist and load them into the local database.

### Species covered

| Common Name | Scientific Name | USDA | Conservation |
|---|---|---|---|
| Coast Live Oak | *Quercus agrifolia* | QUAG | Least Concern |
| Engelmann Oak | *Quercus engelmannii* | QUEN | Vulnerable / CNPS 1B.2 |
| Valley Oak | *Quercus lobata* | QULO | Near Threatened |
| Canyon Live Oak | *Quercus chrysolepis* | QUCH2 | Least Concern |
| Interior Live Oak | *Quercus wislizeni* | QUWI2 | Least Concern |
| California Scrub Oak | *Quercus berberidifolia* | QUBE | Least Concern |
| Island Oak | *Quercus tomentella* | QUTO | Near Threatened |
| Blue Oak | *Quercus douglasii* | QUDO | Least Concern |

## Quick Start — Local Development

### Prerequisites

- Node.js 18+
- Python 3.11+

### 1. Install frontend dependencies

```bash
cd OakProject
npm install
```

### 2. Install backend dependencies

```bash
python -m venv .venv
source .venv/bin/activate        # Windows: .venv\Scripts\activate
pip install -r backend/requirements.txt
```

### 3. Start the backend API

```bash
# From OakProject/ root
uvicorn backend.main:app --reload --port 8008
```

API docs available at http://localhost:8008/docs

### 4. Start the frontend dev server

```bash
npm run dev
```

App available at http://localhost:5173

The Vite dev server proxies `/api/*` requests to the FastAPI backend at
`http://localhost:8008`, so both services must be running for full functionality.

## Fetching Occurrence Data

Run the ETL script to populate the database with real observations from GBIF
and iNaturalist:

```bash
# From OakProject/ root (with virtualenv active)
cd /Users/joshcrasto/Projects/SoCalEcologyProject/OakProject                                                                        
uv run --with-requirements backend/requirements.txt python scripts/fetch-data.py       
```

This will:

1. Create `oaks.db` (SQLite) in the project root if it does not exist.
2. Upsert all 8 species into the database.
3. Fetch up to 300 GBIF occurrence records per species (filtered to the SoCal
   bounding box: lon −120.5 to −114.0, lat 32.5 to 35.8).
4. Fetch up to 200 research-grade iNaturalist observations for species with a
   known taxon ID, plus associated photos.

The script is safe to re-run — duplicate records are silently skipped via the
`(source, source_record_id)` unique constraint.

## Static Export

To export the database contents to static files (useful for hosting without a
live backend):

```bash
# All curated or uncurated observations
python scripts/export-static.py

# Only observations marked as curated
python scripts/export-static.py --curated-only
```

Output:
- `data/occurrences/{species-id}.geojson` — GeoJSON FeatureCollection per species
- `data/photos/{species-id}.json` — array of photo metadata per species

## Production Build

```bash
npm run build
```

Output is in `dist/`. Deploy to any static host (Netlify, GitHub Pages, etc.)
alongside the FastAPI backend (or serve the static GeoJSON files directly and
remove API calls from the frontend).

## API Endpoints

| Method | Path | Description |
|---|---|---|
| GET | `/` | Health check |
| GET | `/species` | List all species |
| GET | `/species/{id}` | Get one species (slug or GBIF key) |
| GET | `/occurrences` | GeoJSON occurrences (bbox, species, limit filters) |
| GET | `/distribution/{species_id}` | GeoJSON for one species |
| GET | `/photos/{species_id}` | Photos for one species |
| POST | `/occurrences/{obs_id}/curate` | Toggle curated flag |
| GET | `/admin/sources` | List data sources with record counts |
| POST | `/admin/fetch/{source}` | Trigger data fetch (stub, returns 202) |
